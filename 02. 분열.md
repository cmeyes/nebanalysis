# 분열 로직

## 1. 개요

분열은 하나의 방울을 두 개의 방울로 나누는 동작이다.

일반적인 질량 증가, 예를 들어 먹이를 먹거나 상대를 흡수하는 경우에는 `targetMass`가 먼저 증가하고 `smoothMass`가 점진적으로 따라간다.
하지만 분열은 이와 다르게 처리된다.

분열은 단순히 목표 질량만 바꾸고 기다리는 방식이 아니라, **부모 셀의 질량과 반지름을 즉시 재설정하고, 새 자식 셀을 즉시 생성하는 동작**이다.

즉, 실제 게임에서 분열 직후 두 방울이 모두 절반 크기로 보이는 것이 정상이다.

---

## 2. 분열 가능 조건

분열은 아무 때나 가능한 것이 아니다.
일반적으로 다음 조건을 만족해야 한다.

| 조건                 | 의미                   |
| ------------------ | -------------------- |
| `smoothMass >= 20` | 현재 반영 질량이 20 이상이어야 함 |
| 셀이 죽은 상태가 아님       | 제거된 셀은 분열 불가         |
| 특정 상태 타이머가 없음      | 분열 제한 상태에서는 불가       |
| 최대 셀 개수에 도달하지 않음   | 셀 개수 제한을 넘을 수 없음     |
| 플레이어가 분열 가능한 상태    | 모드/상태에 따라 제한 가능      |

중요한 점은, **분열 가능 여부는 `targetMass`가 아니라 현재 몸집에 반영된 `smoothMass` 기준**이라는 것이다.

즉, 점수상으로는 20점을 넘었더라도 `smoothMass`가 아직 20에 도달하지 않았다면 분열할 수 없을 수 있다.

---

## 3. 기본 분열 질량 분배

기본 분열은 부모 방울의 질량을 절반으로 나누는 방식이다.

예를 들어 100점 방울이 분열하면 다음과 같다.

| 셀    | 분열 후 질량 |
| ---- | ------: |
| 부모 셀 |      50 |
| 자식 셀 |      50 |

수식으로 쓰면 다음과 같다.
```math
[
m_{\text{child}} = 0.5m_{\text{parent}}
]
```

```math

[
m_{\text{parent,new}} = m_{\text{parent}} - m_{\text{child}}
]
```
기본 분열에서는 결과적으로 부모와 자식이 각각 절반 질량을 가지게 된다.

---

## 4. 분열은 smoothMass 지연의 예외다

먹이를 먹거나 상대를 흡수할 때는 보통 다음 구조를 따른다.

1. `targetMass`가 먼저 변함
2. `smoothMass`가 점진적으로 따라감
3. 반지름이 `smoothMass` 기반으로 갱신됨

하지만 분열은 다르다.

분열 시 부모 셀에는 `addMassPenalty(newMass)`가 호출되는데, 이 함수는 `targetMass`만 바꾸는 것이 아니라 다음 값을 모두 즉시 갱신한다.

| 값            | 처리                     |
| ------------ | ---------------------- |
| `targetMass` | 새 질량으로 즉시 설정           |
| `smoothMass` | 새 질량으로 즉시 설정           |
| `radius`     | `sqrt(newMass)`로 즉시 갱신 |
| `baseSpeed`  | 새 반지름 기준으로 즉시 재계산      |

즉, 부모 셀은 100에서 50으로 서서히 줄어드는 것이 아니라, **즉시 50점 크기로 보정된다.**

자식 셀도 새로 생성될 때부터 분배받은 질량 기준으로 초기화된다.

따라서 실제 게임에서는 100점 셀이 분열하면 다음처럼 처리된다.

| 셀  | targetMass | smoothMass |      radius |
| -- | ---------: | ---------: | ----------: |
| 부모 |         50 |         50 | ($\sqrt{50}$) |
| 자식 |         50 |         50 | ($\sqrt{50}$) |

즉, 분열은 **점진적으로 작아지는 질량 변화**가 아니라, **즉시 절반 질량의 두 셀로 재구성되는 동작**이다.

---

## 5. 새 자식 셀 생성

분열 시 새 자식 셀은 부모 셀의 위치에서 생성된다.

기본 구조는 다음과 같다.

| 항목       | 값                         |
| -------- | ------------------------- |
| 자식 위치    | 부모 셀의 현재 위치               |
| 자식 질량    | 부모 질량의 절반                 |
| 자식 반지름   | ($\sqrt{m_{\text{child}}}$) |
| 자식 이동 방향 | 플레이어가 조준한 방향              |
| 자식 이동 시간 | 자식 반지름 기반으로 결정            |

즉, 자식 셀은 이미 절반 질량으로 초기화된 상태에서 조준 방향으로 튀어나간다.

---

## 6. 분열 후 이동

분열 직후 새 셀은 일정 시간 동안 조준 방향으로 빠르게 이동한다.

코드상 분열 이동 시간은 대략 다음 형태로 계산된다.

[
$t_{\text{split}} = \frac{radius}{80} + 0.5$
]

즉 자식 셀의 반지름이 클수록 분열 후 밀려나가는 시간이 조금 더 길어진다.

예시:

| 자식 질량 |   반지름 | 분열 이동 시간 |
| ----: | ----: | -------: |
|    20 |  4.47 |  약 0.56초 |
|    50 |  7.07 |  약 0.59초 |
|   100 | 10.00 |  약 0.63초 |
|   250 | 15.81 |  약 0.70초 |

분열 직후 자식 셀이 앞으로 튀어나가는 느낌은 이 이동 타이머와 방향값에 의해 만들어진다.

---

## 7. 여러 번 분열할 때

여러 번 분열하면 각 셀은 다시 같은 방식으로 절반씩 나뉜다.

예를 들어 100점 셀 하나가 연속으로 분열하면 다음처럼 된다.

| 단계    | 셀 구성                               |
| ----- | ---------------------------------- |
| 시작    | 100                                |
| 1회 분열 | 50 + 50                            |
| 2회 분열 | 25 + 25 + 25 + 25                  |
| 3회 분열 | 12.5 × 8개, 단 분열 최소 조건 때문에 실제로는 제한됨 |

다만 실제 게임에서는 다음 제한이 있다.

* 셀마다 분열 가능 최소 질량이 필요함
* 최대 셀 개수 제한이 있음
* 분열 직후 상태 타이머나 재결합 관련 제한이 있음

따라서 이론상 무한히 나눌 수 있는 것이 아니라, 일정 조건에서 멈춘다.

---

## 8. 분열 최소 질량

분열 가능 조건 중 가장 중요한 것은 다음이다.

$[
smoothMass \ge 20
]$

즉 현재 반영 질량이 20 이상이어야 분열할 수 있다.

예를 들어 100점 셀은 분열 가능하다.

$[
100 \ge 20
]$

하지만 15점 셀은 분열할 수 없다.

[
15 < 20
]

주의할 점은, 이 기준도 단순히 `targetMass`가 아니라 `smoothMass` 기준이라는 점이다.
따라서 질량이 막 증가한 직후에는 점수상으로는 20을 넘었더라도, `smoothMass`가 아직 따라오지 않았다면 분열이 바로 되지 않을 수 있다.

---

## 9. 분열과 질량 보간의 관계

분열 로직은 질량 시스템에서 중요한 예외다.

일반적인 질량 증가:

| 상황    | 처리 방식                               |
| ----- | ----------------------------------- |
| 먹이 섭취 | `targetMass` 증가 후 `smoothMass`가 따라감 |
| 상대 흡수 | `targetMass` 증가 후 `smoothMass`가 따라감 |
| 홀 보상  | `targetMass` 증가 후 `smoothMass`가 따라감 |

분열:

| 상황         | 처리 방식                                       |
| ---------- | ------------------------------------------- |
| 부모 셀 질량 감소 | `targetMass`, `smoothMass`, `radius` 즉시 재설정 |
| 자식 셀 생성    | 분배받은 질량으로 즉시 초기화                            |
| 분열 후 이동    | 별도 이동 타이머로 처리                               |

따라서 분열은 “부드럽게 크기가 변하는 질량 변화”라기보다는, **셀 구조를 즉시 재구성하는 동작**이다.

---

## 10. DASH 모드의 예외

일부 모드에서는 분열 버튼이 일반 분열이 아니라 다른 행동으로 작동한다.

대표적으로 DASH 모드에서는 셀이 둘로 나뉘지 않고, 기존 셀이 조준 방향으로 돌진한다.

DASH에서는 대략 다음 처리가 이루어진다.

| 항목     | 처리         |
| ------ | ---------- |
| 셀 개수   | 증가하지 않음    |
| 이동     | 조준 방향으로 돌진 |
| 질량     | 일정 비율 감소   |
| 기본 감소량 | 약 10%      |

즉 DASH 모드에서 분열 입력은 “셀 나누기”가 아니라 “질량을 소모하고 돌진하기”에 가깝다.

일반 분열 로직과는 별도 예외로 보는 것이 좋다.

---
## 11. 핵심 요약

* 분열은 하나의 셀을 두 개의 셀로 나누는 동작이다.
* 기본 분열은 질량을 50% / 50%로 나눈다.
* 분열 가능 여부는 `smoothMass >= 20` 조건을 따른다.
* 자식 셀은 분배받은 질량으로 즉시 생성된다.
* 부모 셀도 `targetMass`, `smoothMass`, `radius`, `baseSpeed`가 즉시 재설정된다.
* 따라서 분열 직후 두 셀은 모두 절반 크기로 보이는 것이 정상이다.
* 분열은 일반적인 질량 보간과 달리, 셀 구조를 즉시 재구성하는 특수 동작이다.
